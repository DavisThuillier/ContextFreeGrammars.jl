API Reference

Semirings

ContextFreeGrammars.AbstractSemiring — Type
AbstractSemiring

Abstract supertype for the semirings used to weight grammar productions and parse values. A semiring $(K, ⊕, ⊗, 0̄, 1̄)$ supplies the algebraic structure over which cyk accumulates inside values.

Each concrete semiring is a singleton type paired with an element type that subtypes AbstractSemiringElement. To define a custom semiring, create both types and implement element_type, semiring, Base.:+, Base.:*, Base.zero, Base.one, and—if unit-production elimination is needed—star for the element type.

See also BooleanSemiring, ProbabilisticSemiring, CountSemiring.

source
ContextFreeGrammars.AbstractSemiringElement — Type
AbstractSemiringElement

Abstract supertype for the elements (weights) of an AbstractSemiring.

A concrete subtype wraps a single value, retrievable with val, and must implement Base.:+ (semiring addition ⊕), Base.:* (semiring multiplication ⊗), Base.zero (the additive identity 0̄), and Base.one (the multiplicative identity 1̄). A star method may also be supplied for the closure operations used when removing unit productions (see remove_unit_productions).

source
ContextFreeGrammars.BooleanElement — Type
BooleanElement(val::Bool)

An element of the BooleanSemiring. Addition is logical OR, multiplication is logical AND, zero is false, and one is true.

source
ContextFreeGrammars.BooleanSemiring — Type
BooleanSemiring()

The Boolean semiring $(\{0,1\}, ∨, ∧, 0, 1)$ whose elements are BooleanElements. Parsing under this semiring answers the recognition question: val of the resulting chart is true exactly when the input is in the language of the grammar. It is the default semiring throughout the package.

source
ContextFreeGrammars.CountElement — Type
CountElement(val::Union{Int,InfInt})

An element of the CountSemiring, holding a non-negative integer count or the sentinel ∞. Addition and multiplication are the usual integer + and ×, zero is 0, and one is 1. The closure star(a) is one when a is zero and ∞ otherwise.

source
ContextFreeGrammars.CountSemiring — Type
CountSemiring()

The counting semiring $(ℕ ∪ \{∞\}, +, ×, 0, 1)$ whose elements are CountElements. Parsing under this semiring makes val of the chart the number of distinct derivations of the input, which is reported as ∞ when the grammar admits infinitely many.

source
ContextFreeGrammars.ProbabilisticElement — Type
ProbabilisticElement(val::Float64)

An element of the ProbabilisticSemiring. Addition and multiplication are the usual real + and ×, zero is 0.0, and one is 1.0. The closure star(a) is 1 / (1 - a).

source
ContextFreeGrammars.ProbabilisticSemiring — Type
ProbabilisticSemiring()

The probabilistic (inside) semiring $([0, ∞), +, ×, 0, 1)$ whose elements are ProbabilisticElements. Used with a weighted grammar (e.g. a PCFG), parsing under this semiring makes val of the chart the total probability mass of all derivations of the input.

Context-free grammars

ContextFreeGrammars.ContextFreeGrammar — Type
ContextFreeGrammar{N, T, E}

A weighted context-free grammar with nonterminals of type N, terminals of type T, and production weights of semiring element type E. Its fields are the sets of nonterminals and terminals, the vector of productions, and the start symbol.

Constructors

ContextFreeGrammar(productions, start; semiring = BooleanSemiring())
ContextFreeGrammar(nonterminals, terminals, productions, start; semiring = BooleanSemiring())

Build a grammar from a collection of productions. Each production is a tuple (lhs, rhs)—or (lhs, rhs, weight) for a non-Boolean semiring where lhs is a nonterminal and rhs is a vector of symbols. In the two-argument form the terminal and nonterminal alphabets are inferred from the productions; in the four-argument form they are given explicitly.

Examples

G = ContextFreeGrammar(
    [(:S, [:NP, :VP]), (:NP, ["fish"]), (:VP, ["swim"])],
    :S,
)

ContextFreeGrammars.remove_useless_symbols — Method
remove_useless_symbols(cfg::AbstractGrammar)

Return a new ContextFreeGrammar equivalent to cfg with all useless symbols removed. A symbol is useful only if it is both generating and reachable. Non-generating symbols (and the productions mentioning them) are dropped first, then unreachable symbols are dropped from the result; the order matters, so that reachability is assessed on the already-pruned grammar.

Chomsky Normal Form

ContextFreeGrammars.ChomskyNormalFormContextFreeGrammar — Type
ChomskyNormalFormContextFreeGrammar{N,T,E}

A context-free grammar in Chomsky normal form: every production is either $A ⇒ a$ (a single terminal) or $A ⇒ B C$ (two nonterminals). This is the form required by cyk. It shares the field layout and accessors of ContextFreeGrammar.

Constructors

ChomskyNormalFormContextFreeGrammar(cfg::ContextFreeGrammar)
ChomskyNormalFormContextFreeGrammar(productions, start; semiring = BooleanSemiring())

Convert an existing grammar (or one built inline from productions and start) to Chomsky normal form, preserving the weight of every derivation. The conversion follows Hopcroft & Ullman, Introduction to Automata Theory, Languages, and Computation (1979), pp. 92–94: unit productions are eliminated (see remove_unit_productions), useless symbols are removed (see remove_useless_symbols), terminals occurring in long bodies are abstracted into fresh nonterminals, and remaining long bodies are binarized.

Conversion mints fresh nonterminals of type N; methods are provided for N === Symbol and N === String. To convert grammars over another nonterminal type, define fresh_nonterminal(::Type{N}, V, Σ; prefix).

source
ContextFreeGrammars.in_chomsky_normal_form — Method
in_chomsky_normal_form(production::Production)
in_chomsky_normal_form(cfg::AbstractGrammar)

Test whether a single production is in Chomsky normal form—i.e. its body is either one terminal ($A ⇒ a$) or two nonterminals ($A ⇒ B C$)—or whether every production of a grammar cfg is. A ChomskyNormalFormContextFreeGrammar satisfies this by construction.

source
ContextFreeGrammars.remove_unit_productions — Method
remove_unit_productions(productions)

Return a new vector of Productions equivalent to productions but free of unit productions (rules of the form $A ⇒ B$ with a single nonterminal body). The non-unit productions are kept, and for every pair of nonterminals connected through a chain of unit productions the corresponding non-unit productions are re-attached with the chain's accumulated weight. The chain weights are obtained from the transitive closure of the unit-production graph, computed over the grammar's semiring (using its star operation), so cycles of unit productions are handled correctly.

CYK parsing

ContextFreeGrammars.CYKParseChart — Type
CYKParseChart{N, T, E}

The dynamic-programming chart produced by cyk. For each span (i, j) of the input and each nonterminal A, it stores the inside value—the total semiring weight of all derivations of input[i+1:j] from A—as an element of type E. Spans use half-open, zero-based indexing, so the whole input is the span (0, length(input)).

Read inside values with inside and the overall value of the input with val. Access the chart's metadata with input, nonterminals, start_index, and start.

source
ContextFreeGrammars.cyk — Method
cyk(input::AbstractVector, grammar::AbstractGrammar)
cyk(input::String, grammar::AbstractGrammar; dlm = " ")

Run the CYK algorithm and return the filled CYKParseChart for input under grammar. The grammar must be in Chomsky normal form (see in_chomsky_normal_form and ChomskyNormalFormContextFreeGrammar), otherwise an ArgumentError is thrown.

This is the semiring formulation of CYK (Goodman, Semiring Parsing): every chart cell accumulates the inside value of its span in the grammar's semiring, so the same code recognizes (Boolean), counts derivations (CountSemiring), or sums their probability mass (ProbabilisticSemiring) depending on the production weights.

When input is a String it is split into tokens on the delimiter dlm before parsing. Query the resulting chart with inside or val.

source

ContextFreeGrammars.inside — Method
inside(i::Int, j::Int, C, chart::AbstractParseChart)
inside(i::Int, j::Int, k::Int, chart::CYKParseChart)
inside(subsequence::AbstractVector, C, chart::AbstractParseChart)
inside(substring::AbstractString, C, chart::AbstractParseChart; dlm = " ")
inside(subsequence::AbstractVector, C, input::AbstractVector, grammar::AbstractGrammar)

Return the inside value for nonterminal C over a span of the input recorded in chart—that is, the total semiring weight of all derivations of that span from C, unwrapped with val to a plain value.

The span may be given directly as the half-open, zero-based bounds (i, j), with the nonterminal named either by its symbol C or by its column index k. Alternatively, pass a subsequence (or whitespace/dlm-delimited substring) and the span is located within input(chart) by its first occurrence, throwing an ArgumentError if it is not present. The final form takes a raw input and grammar, parsing with cyk before querying.

source

source
ContextFreeGrammars.val — Method
val(chart::AbstractParseChart)
val(input::AbstractVector, grammar::AbstractGrammar)
val(input::String, grammar::AbstractGrammar; dlm = " ")

Return the value of the whole input under the grammar: the inside value of the start symbol over the entire span, unwrapped with val. Its meaning depends on the semiring—true/false for recognition with the BooleanSemiring, a derivation count with the CountSemiring, or total probability mass with the ProbabilisticSemiring.

Given a filled CYKParseChart it reads the answer off the chart; given a raw input (a vector of terminals, or a dlm-delimited String) and a grammar, it first parses with cyk.
